# 🚽 RobloxSkibidi TD

Командный tower defense для Roblox в стилистике skibidi toilet:
волны врагов-камикадзе, боссы-вертолёты, магазин башен с режимами атак,
прокачка, ремонт и киберпанк-интерфейс.

## 🎮 Как играет игрок

1. Получает стартовые кредиты ₵ (кошелёк сохраняется между сессиями).
2. Покупает башни в магазине снизу экрана и ставит их на карту.
3. Волны врагов идут по маршрутам к базе; дошедший враг взрывается
   об базу как камикадзе.
4. Между делом: качает башни по уровням, чинит их после боссов,
   переключает режимы, продаёт лишнее.
5. На волнах с боссами — предупреждение «⚠ ПРИБЛИЖАЕТСЯ», потом
   вертолёт кружит вокруг башен и глушит/сносит их.
6. Победа после последней волны или поражение при гибели базы —
   и кнопка «⟳ ИГРАТЬ СНОВА» для нового раунда.

## ⚙️ Стек

- **Luau** — весь геймплейный код
- **[Rojo](https://rojo.space) 7.6.1** — синхронизация файлов с Roblox Studio
- **[Rokit](https://github.com/rojo-rbx/rokit)** — менеджер инструментов
- **luau-lsp** — статический анализ перед коммитом

## 🚀 Быстрый старт

```bash
git clone https://github.com/Maramazzi/rsProjectSkibidi.git
cd rsProjectSkibidi
rokit install            # ставит rojo из манифеста проекта
rojo serve               # поднимает сервер синхронизации
```

Дальше в Roblox Studio: **Plugins → Rojo → Connect** — код из `src/`
появится в нужных местах. Запускай Play и играй.

> ⚠️ Чтобы сохранение денег (DataStore) работало в Studio:
> **Game Settings → Security → Enable Studio Access to API Services**.
> Без этого все просто начинают со стартовой суммы — ошибок не будет.

Полная инструкция по установке с нуля (плагин Studio, VS Code,
анализатор) — [DOCUMENTATION.md §1–2](DOCUMENTATION.md).

## ✨ Что уже в игре

**Башни** (8 видов): винтовка, огнемёт (конус), лазер (разгон урона),
ракетница (сплеш), клинок, звуковая, гибрид (свитч режимов), вербовщик
(делает врагов союзниками на время).

- **Уровни**: 2 апгрейда у каждой башни — урон/радиус/магазин растут,
  цена каждого уровня выше; при продаже возвращается доля всех вложений
- **HP, ремонт и реген**: боссы сносят башни; вне боя башня медленно
  чинится сама, за деньги — мгновенно
- **Магазин и перезарядка**: у дальнобойных башен ограниченный магазин
- **Обзор**: враги за зданиями (`workspace.Buildings`) невидимы для башен
- **Размещение через UI**: призрак башни, круг радиуса, валидация на сервере

**Враги и боссы**:

- Волны с множителями HP/скорости, стан-врагами (глушат башни)
  и событиями карты (открытие/закрытие маршрутов)
- Боссы-вертолёты по шаблонам: летают вне маршрутов с физикой полёта
  (крен, тангаж, инерция), кружат вокруг башен, отдыхают между заходами;
  поддерживают свои модели из ServerStorage (см. DOCUMENTATION §6.1)
- Перевербованные союзники: турель или рукопашник, живут 30 секунд

**Сервис**:

- **Экономика**: кредиты ₵, награды за убийства и боссов,
  кошелёк игрока сохраняется в DataStore между сессиями
- **Рестарт**: после победы/поражения любой игрок жмёт «Играть снова» —
  «чистое поле»: купленные башни убираются с возвратом 100% вложений,
  студийные восстанавливаются, кошелёк остаётся
- **Лобби** (отдельный place): кнопка «✓ ГОТОВ», отсчёт и телепорт
  всей компании в игровой плейс — настройка в DOCUMENTATION §12
- **Эффекты на клиенте**: лучи, взрывы, циферки урона
- **Мобильная поддержка**: тач-размещение и тап-меню башни
- **Киберпанк-UI**: общая тема в одном модуле (`Theme.luau`)

## 📁 Структура

```
src/
├── shared/Classes/    → ReplicatedStorage.Classes
│   ├── TowerClass       конфиги башен, уровни, бой, ремонт/реген
│   ├── EnemyClass       враги: движение, камикадзе, стан, вербовка
│   ├── BossClass        боссы-вертолёты: полёт, орбита, атаки
│   ├── BaseClass        главная база: HP, рестарт
│   ├── Effects          боевые эффекты (сервер шлёт, клиент рисует)
│   └── PlacementZones   зоны и дистанции размещения
├── server/            → ServerScriptService.Server
│   ├── GameManager      волны, спавн, маршруты, рестарт, GameData
│   ├── TowerService     покупка/продажа/апгрейд/ремонт, валидация
│   ├── Economy          деньги + сохранение в DataStore
│   └── TestMap          автогенерация тестовой локации
├── client/            → StarterPlayerScripts.Client
│   ├── TowerShopUI      магазин + размещение
│   ├── TowerInfoUI      меню башни (статы, режимы, апгрейд, ремонт)
│   ├── AdminUI          HUD, баннеры босса и конца игры, рестарт
│   ├── EffectsClient    приёмник эффектов
│   └── Theme            киберпанк-тема интерфейса
└── lobby/             → ЛОББИ-плейс (свой проект: lobby.project.json)
    ├── server/          готовность игроков, отсчёт, телепорт
    └── client/          UI лобби
```

Геометрия карты, модели и зоны живут в Studio — Rojo синхронизирует
только код (partial sync, см. `default.project.json`). Лобби-плейс
синкается отдельно: `rojo serve lobby.project.json`.

## ✅ Перед коммитом

```bash
rojo sourcemap default.project.json -o sourcemap.json
luau-lsp analyze --definitions=<globalTypes> --sourcemap=sourcemap.json src
rojo build -o test.rbxlx                               # собирается?
```

Главные правила (подробно — [HANDBOOK.md §2.5](HANDBOOK.md)):
1. Источник правды — файлы репозитория; скрипты в Studio не редактируем.
2. Сервер авторитетен: клиент шлёт намерения, сервер всё перепроверяет.
3. Баланс — только в конфиг-таблицах.
4. Не коммитим `*.rbxl(x)`, `sourcemap.json`, бэкапы.
