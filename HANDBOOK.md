## 1.0 Что установить на компьютер (один раз)

Прежде чем что-то делать, поставь три программы:

1. **Roblox Studio** — сам редактор игры. Скачивается с
   [create.roblox.com](https://create.roblox.com) (кнопка Start Creating).
   Войди под своим аккаунтом Roblox.
2. **Visual Studio Code (VS Code)** — в нём редактируется КОД проекта.
   Скачать: [code.visualstudio.com](https://code.visualstudio.com).
3. **Rokit** — менеджер инструментов, он поставит Rojo нужной версии.
   Инструкция установки: [github.com/rojo-rbx/rokit](https://github.com/rojo-rbx/rokit)
   (скачай установщик из раздела Releases, запусти).

> Если на любом шаге что-то не ставится / ругается — копируй текст ошибки и неси мне.

✅ **Как проверить:** открой PowerShell (меню Пуск → напиши «PowerShell» → Enter)
и набери `rokit --version`. Если показалась версия — Rokit стоит.

## 1.1 Мини-ликбез по интерфейсу Studio (прочитай один раз)

Открой файл проекта `Roblox.rbxl` (двойной клик по нему, либо в Studio
File → Open from File).

**Главные окна** (если какого-то нет на экране — включи во вкладке **View** сверху):
- **Explorer** (View → Explorer) — дерево всех объектов игры. Здесь ты проводишь
  90% времени. Важные ветки внутри:
  - `Workspace` — всё, что физически на карте (земля, база, враги, башни);
  - `ServerStorage` — «склад» сервера, игроки его не видят (там лежит шаблон врага);
  - `ReplicatedStorage` — общий «склад» (туда Rojo кладёт папку `Classes`).
- **Properties** (View → Properties) — свойства ВЫДЕЛЕННОГО объекта (имя, размер,
  цвет, галочки). Кликнул объект в Explorer → тут его настройки.
- **Output** (View → Output) — окно сообщений. Сюда падают `print(...)` из кода
  и **красные ошибки**. Если что-то не работает — первым делом смотри сюда.

**Базовые действия в Explorer:**
- **Выделить объект** — клик. **Раскрыть/свернуть ветку** — стрелочка ► слева от имени.
- **Создать объект внутри другого** — наведи курсор на родителя, появится кнопка
  **⊕** справа от имени → клик → выбери тип (`Folder`, `Part` и т.д.).
  Или: правый клик по родителю → **Insert Object**.
- **Переименовать** — двойной клик по имени (или выдели и нажми **F2**).
  ⚠️ Имена важны до буквы и пробела: `skibidi toilet` ≠ `Skibidi Toilet` ≠ `skibiditoilet`.
- **Переместить объект к другому родителю** — перетащи мышкой в Explorer
  (drag-and-drop), либо вырежи **Ctrl+X** и вставь в нужного родителя **Ctrl+V**.
- **Скопировать** — **Ctrl+C / Ctrl+V**, или «дублировать на месте» — **Ctrl+D**.
- **Удалить** — выдели и нажми **Delete**.

**Камера в Viewport (большое 3D-окно):**
- зажми **правую кнопку мыши** и води — крутишь взгляд;
- **W A S D** — летать вперёд/влево/назад/вправо; **E / Q** — вверх/вниз;
- колесо мыши — приблизить/отдалить;
- выдели объект в Explorer и нажми **F** — камера подлетит прямо к нему (очень удобно искать).

**Инструменты для деталей** (вкладка **Model** сверху): **Move** (двигать),
**Scale** (менять размер), **Rotate** (вращать). Выбрал инструмент → у объекта
появятся стрелочки/кубики, тяни их мышкой.

**Как поставить АТРИБУТ объекту** (понадобится не раз):
1. Выдели объект в Explorer.
2. В окне **Properties** прокрути в самый низ — там секция **Attributes**.
3. Жми **⊕ Add Attribute** → впиши имя (например `Enabled`) → выбери тип
   (`Boolean` = да/нет, `Number` = число, `String` = текст) → ОК.
4. Появится строка-поле — поставь галочку или впиши значение.

**Как запустить игру для проверки:** вкладка **Home** или **Test** → кнопка
**▶ Play** (побегать персонажем) или **Run** (запустить мир без персонажа).
Остановить — **■ Stop** (там же).

## 1.2 🟦 Подключить Rojo — без этого скриптов в игре НЕ будет

Скрипты живут в файлах проекта, а в Studio попадают через программу Rojo.
Делается так:

1. Открой папку проекта в терминале:
   - проще всего — открой папку проекта в **VS Code** (File → Open Folder →
     выбери папку `RobloxSkibidi`), затем вверху меню **Terminal → New Terminal**.
     Терминал откроется уже в нужной папке.
2. В терминале выполни по очереди (каждую строку — Enter, дождись завершения):
   ```powershell
   rokit install          # поставит Rojo нужной версии из манифеста проекта
   rojo plugin install    # установит плагин Rojo прямо в твой Studio
   rojo serve             # запустит синхронизацию
   ```
3. После `rojo serve` **не закрывай это окно терминала** — пока оно работает,
   идёт синхронизация.
4. Переключись в Studio: верхняя панель → вкладка **Plugins** → иконка **Rojo**
   → кнопка **Connect**.

✅ **Как проверить:** после Connect в Explorer САМИ появятся скрипты —
в `ServerScriptService` папка `Server`, в `ReplicatedStorage` папка `Classes`,
в `StarterPlayer → StarterPlayerScripts` папка `Client`. Если появились — Rojo работает.

> Пока `rojo serve` запущен и Studio подключён — любая правка кода в VS Code
> мгновенно прилетает в Studio. Решение проблем подключения — DOCUMENTATION §1–2.

## 1.3 🟦 Удалить старые дубли скриптов — САМОЕ ВАЖНОЕ ⚠️

Раньше код писали прямо в Studio, и старые копии скриптов могли остаться в файле.
Их надо **удалить**, иначе логика выполнится ДВАЖДЫ (двойной урон, два спавна
врагов, странные глюки).

Пошагово:
1. В Explorer раскрой по очереди эти ветки и поищи в них «лишнее»:
   - `ServerScriptService` — оставляем только папку **`Server`** (её принёс Rojo).
     Всё остальное (старые `GameManager`, `Server` с «hello world») — удалить.
   - `StarterPlayer → StarterPlayerScripts` — оставляем только папку **`Client`**.
     Старый `AdminUI` и прочее — удалить.
   - `ReplicatedStorage` — оставляем папку **`Classes`** (от Rojo). Старые папки
     `Classes`/`Shared`, лежавшие тут ДО первого Connect, — удалить.
   - `StarterGui` — если там лежат старые UI-скрипты, удалить.
2. Любые `Script` / `LocalScript` с `print("Hello world")` или непонятного
   назначения — удаляй (выдели → Delete).

> 🔎 **Как точно отличить «своё от Rojo» от «старого руками»:** нажми в плагине
> Rojo кнопку **Disconnect**. Всё, что принёс Rojo, ИСЧЕЗНЕТ. Значит, то, что
> ОСТАЛОСЬ в перечисленных местах, — старое, его и удаляй. Потом снова **Connect**.

✅ **Как проверить:** в `ServerScriptService` ровно одна папка `Server`,
в `StarterPlayerScripts` ровно одна папка `Client`, дублей нет.

## 1.4 🟦 Включить доступ к API (чтобы деньги сохранялись)

Без этого DataStore (сохранение кошелька между сессиями) молча не работает —
все игроки будут начинать с 400💰 каждый раз.

1. Вкладка **Home** (или меню **File**) → **Game Settings**.
2. В открывшемся окне слева выбери раздел **Security**.
3. Включи тумблер **Enable Studio Access to API Services** (станет синим/зелёным).
4. Нажми **Save** внизу.

✅ **Как проверить:** запусти Play, заработай денег, останови, запусти снова —
сумма должна сохраниться (а не сброситься на 400). В Output не должно быть
красных ошибок про DataStore.

> Это нужно ТОЛЬКО для проверки сохранения в Studio. В опубликованной игре
> DataStore работает и без этой галочки. Не хочешь возиться — пропусти: бои
> проверяются и так, просто деньги не будут сохраняться.

## 1.5 🟦 Проверить ГЛАВНУЮ БАЗУ (GeneralTower)

База — то, что защищают игроки. Код ищет её строго по имени.

1. В **Workspace** найди модель (Model) с именем ровно **`GeneralTower`**.
   - Нет такой? Создай: правый клик по `Workspace` → Insert Object → **Model**,
     переименуй в `GeneralTower` (F2). Внутрь положи хотя бы одну `Part`
     (правый клик по модели → Insert Object → Part) — это и будет корпус базы.
2. Главную деталь корпуса лучше назвать **`TowerBase`** (код сначала ищет деталь
   с таким именем; если не нашёл — берёт любую первую Part внутри модели).
3. Выстави модели **PrimaryPart** (это «якорь» модели):
   - выдели модель `GeneralTower` в Explorer;
   - в **Properties** найди строку **PrimaryPart** → кликни в пустое поле напротив;
   - теперь кликни в Explorer на нужную деталь (`TowerBase`) — её имя впишется.

✅ **Как проверить:** запусти Play — над базой висит полоса HP `⚡ ГЛАВНАЯ БАЗА
10000 / 10000`. Если в Output `GeneralTower: не найдена основная часть!` — внутри
модели нет ни одной Part (см. шаг 1).

## 1.6 🟦 Проверить модель ВРАГА (skibidi toilet)

1. Найди модель **`skibidi toilet`** (имя строго маленькими буквами, с пробелом).
   Она может лежать в **Workspace** или в **ServerStorage** — код сам перенесёт
   её в ServerStorage и будет клонировать из неё врагов.
2. Выстави ей **PrimaryPart** (обычно деталь `HumanoidRootPart`):
   - выдели модель `skibidi toilet` в Explorer;
   - в **Properties** найди строку **PrimaryPart** → кликни в пустое поле напротив;
   - кликни в Explorer на `HumanoidRootPart` (или главную деталь тела) — имя впишется.

✅ **Как проверить:** запусти Play — через несколько секунд по дороге пойдут
враги. Если в Output `Модель 'skibidi toilet' не найдена...` — имя написано
неправильно или модели нет. Если враги «разъезжаются» с полосой HP — не выставлен
PrimaryPart (шаг 2).

## 1.7 🟦 Проверить МАРШРУТ (WaypointsFolder)

Без маршрута враги не знают, куда идти, и **код будет вечно ждать эту папку**
(игра словно «зависнет» на старте, врагов не будет).

1. В **Workspace** должна быть папка (Folder) с именем ровно **`WaypointsFolder`**.
   Нет — создай: правый клик по `Workspace` → Insert Object → **Folder** → переименуй.
2. Положи внутрь точки маршрута — это обычные `Part`:
   - создай Part (вкладка **Home** → кнопка **Part**), перетащи его в Explorer
     внутрь `WaypointsFolder`;
   - переименуй с цифрой: `Waypoint1` (или просто `1`);
   - инструментом **Move** поставь его в начале дороги (где появляются враги);
   - **дублируй** (Ctrl+D), переименуй в `Waypoint2`, передвинь дальше по дороге;
   - так повтори по всей дороге до базы (`Waypoint3`, `4`, … — сколько нужно).
3. Точки нумеруй по порядку движения. Код сортирует их по числу в имени и сам
   делает невидимыми/проходимыми — красить и якорить не обязательно.

✅ **Как проверить:** запусти Play — враги идут по точкам от спавна к базе.
В Output должна быть строка `[TestMap] Создано 2 маршрута...` (см. ниже).

> **Второй маршрут руками делать НЕ обязательно.** Модуль `TestMap` при старте
> сам построит из твоих точек два пути: `Route1` (прямой) и `Route2` (дуга вбок),
> враги выбирают случайный. Выключить автогенерацию: `TestMap.ENABLED = false`
> в файле `src/server/TestMap.luau`.

## 1.8 🟨 (По желанию) Зоны размещения башен — PlacementZones

Это площадки, где игрокам МОЖНО ставить башни. **Нет этой папки = ставить можно
где угодно** — для теста полностью ок, можешь пропустить.

1. В **Workspace** создай папку (Folder) с именем ровно **`PlacementZones`**.
2. Создай плоский `Part` (вкладка Home → Part), инструментом **Scale** расплющь
   его в «коврик» и положи на площадку, где разрешено строить. Перетащи этот Part
   в Explorer внутрь `PlacementZones`.
3. Выдели этот Part и в **Properties** поставь:
   - **Anchored** = ✅ (галочка) — чтобы не падал;
   - **CanCollide** = ☐ (снять галочку) — чтобы не мешал;
   - **Transparency** = `0.5` (полупрозрачный) или `1` (невидимый) — на вкус.
4. Накопируй (Ctrl+D) таких ковриков на все разрешённые площадки.

Башню разрешат поставить, только если её основание попадает внутрь одного из
этих Part'ов. Плюс между башнями всегда минимум 6 стадов (это уже в коде).

✅ **Как проверить:** в Play возьми башню из магазина — круг радиуса **зелёный**
над зоной и **красный** вне зоны; вне зоны поставить нельзя.

## 1.9 🟨 (По желанию) Здания, закрывающие обзор — Buildings

Башня НЕ стреляет по врагу, если между ними стоит «здание». **Нет папки = обзор
ничем не закрыт** — можешь пропустить.

1. В **Workspace** создай папку (Folder) с именем ровно **`Buildings`**.
2. Создавай любые `Part` (стены, дома, кубы) и складывай их в эту папку,
   ставя между дорогой и местами башен. Любой Part в папке `Buildings`
   перекрывает линию обзора.

✅ **Как проверить:** поставь башню за таким зданием — враг за ним НЕ
обстреливается, хоть и в радиусе; вышел из-за здания — огонь возобновился.

## 1.10 🟨 (По желанию) Поставить башни прямо на карте (Studio-башни)

Можно заранее разложить башни на карте — они активируются сами и будут «ничьи»
(продать нельзя, но чинить/переключать можно).

1. Найди в **Workspace** модель-камеру **`Rust`** (она же служит шаблоном магазина).
2. Скопируй её (Ctrl+C → Ctrl+V) и инструментом **Move** поставь куда нужно.
3. Скажи коду, какой это тип башни — задай копии **атрибут** `TowerType` (тип
   `String`, как ставить атрибут — §1.1) со значением одного из ключей:
   `Rust`, `RustFlamer`, `RustLaser`, `RustRocket`, `RustMelee`, `RustSonic`,
   `RustHybrid`, `RustConverter`.
   (Если просто назвать модель именем типа, например `RustLaser`, — тоже сработает.)

✅ **Как проверить:** в Play башня стоит на карте и сама стреляет по врагам;
при наведении показывает «Башня с карты».

## 1.11 🟨 (Когда будут готовы) Модели боссов-вертолётов

Сейчас боссы — заглушки из кубиков. Когда художник отдаст модель:
1. Положи её в **ServerStorage** с именем `Boss_StunCopter` или
   `Boss_DestroyerCopter` — подхватится сама вместо заглушки.
2. Правила сборки модели:

   - деталь с именем `Muzzle` — точка вылета снарядов (нет — стреляет из корпуса).
3. Подробно про сборку — DOCUMENTATION.md §6.1.

## 1.13 ✅ Прогнать тест-сценарий (после Connect и Play)

Нажми **▶ Play** и проверь по порядку:

1. Слева сверху — счётчик `💰 400`.
2. Внизу — панель магазина с кнопками башен.
3. Купи «Винтовку» → призрак ходит за курсором, круг **зелёный** в зоне / **красный** вне.
4. Поставь 2 башни рядом — вторая не встанет ближе 6 стадов (круг покраснеет).
5. Враги идут по маршруту, башни стреляют, **циферки урона** всплывают над врагами.
6. За убитых врагов всем +25💰 (за самоподрыв врага о базу — награды нет).
7. Наведи курсор на башню → панель справа внизу: статы + круг радиуса.
8. Купи «Гибрид» (400) → кнопка «⇄ Сменить режим: Залп ракет» → переключи,
   статы и радиус должны смениться.
9. **Ракеты неуправляемые**: ракетница / «Залп ракет» бьют в точку, где враг
   БЫЛ в момент выстрела. По медленным попадают, быстрые (волны с `speedMult`)
   могут уходить из-под взрыва — это нормально, так задумано.
10. Продай свою башню → вернётся +70% вложений, башня исчезла.
11. Доиграй до конца → баннер «🏆 ПОБЕДА!» (или дай базе умереть → «💀 ПОРАЖЕНИЕ»);
    кнопка «⟳ Играть снова» сбрасывает поле, кошелёк остаётся.
12. **Мобильный тест**: вкладка **Test** → **Device** → выбери телефон.
    Покупка: кнопка → тап по карте → кнопки «✅ Поставить / ❌ Отмена».
    Меню башни — по тапу.

**Проверки по боссам/механикам:**

### 2.5 Архитектурные законы проекта (нарушать нельзя)

1. **Сервер авторитетен.** Урон, деньги, постановка, продажа, свитч — только сервер.
   Клиент шлёт «намерения» через ремоуты, сервер ВСЁ перепроверяет
   (тип, цену, зону, дистанцию, владельца, рейт-лимит). Новые ремоуты — по тому же
   шаблону, см. `TowerService.setupRemotes()`.
2. **Никакого визуала на сервере.** Разовые эффекты (лучи, взрывы, циферки) —
   только через `Effects.beam/sphere/damageNumber` — их рисует клиент.
   Сервер не делает `Instance.new("Part")` для красоты. Исключения уже
   задокументированы в шапке `Effects.luau` (лазер, ракета).
3. **Баланс — только в конфигах.** Урон/цены/HP/волны меняются в таблицах
   (`TOWER_CONFIGS`, `WAVES`, константы `Economy`), не внутри логики.
4. **Деньги — только через `Economy.spend/give/rewardAll`.** Никаких прямых
   `money.Value = ...` в другом коде.
5. **У врагов и эффектов `CanQuery = false`** — чтобы курсор и рейкасты их
   игнорировали. Новые «нефизические» части — тоже.

### 2.6 Частые задачи — рецепты

**Добавить башню/скин:** конфиг в `TOWER_CONFIGS` + ключ в `SHOP_ORDER`
(`TowerClass.luau`) — всё, кнопка в магазине и шаблон появятся сами.
Формат конфига — DOCUMENTATION.md §5.3.

**Дать башне переключаемые режимы:** добавь ей массив `modes = {...}`
(пример — RustHybrid). Кнопка свитча в меню появится автоматически.

**Изменить волны/сложность:** таблица `WAVES` в `GameManager.server.luau`:
`count`, `spawnDelay`, `pauseAfter`, `hpMult`, `speedMult`, `mapEvent`.
Волна заканчивается, когда убиты все её враги.

**Добавить босса:** конфиг в `BOSS_CONFIGS` (`BossClass.luau`) + имя в `bosses = {...}`
нужной волны в `WAVES`. `stunDuration > 0` — глушит башни, `damage > 0` — сносит их HP,
`weapon = "laser" | "rocket"` — тип снаряда. Своя модель: положи её в ServerStorage
и укажи `templateName = "ИмяМодели"` в конфиге (иначе заглушка-вертолёт из Part'ов).

**Сделать врагов волны стан-врагами:** `stunners = N` в таблице `WAVES` —
N случайных врагов волны получат жёлтый шар и будут глушить башни.

**Дать башне перезарядку:** `magazine = N, reloadTime = T` в её конфиге.
Убрать перезарядку — удали эти поля (по умолчанию боезапас бесконечный).

**Новый вид врага:** сейчас один шаблон. Расширение: добавить модель в
ServerStorage и поле `template` в WAVES → передать в `spawnEnemy`. Согласуй с Ильёй.

**Кат-сцена при смене карты (задача 2.1):** клиентский скрипт подписывается на
`ReplicatedStorage.MapChanged.OnClientEvent` — получишь `{open={...}, close={...}}`.

**Заменить заглушечный визуал на модели (задача 4, Чурюмов):**
- модели башен → положить в `ServerStorage`/попросить добавить выбор в
  `buildTowerTemplates()` (TowerService);
- эффекты выстрелов → редактируются функции `Effects.render*` (клиент!) и
  `VISUALS.*` в TowerClass (там только вызовы Effects).

### 2.7 Тестирование (Данзан)
